# MID POP ワンボタン・パニック! プロトタイプ

企画書の全フロー(スロット→リーチ→ミニゲーム4種+ボーナス+復活チャレンジ→結果)を、仮素材(色付き図形+シンセ音)で動かすプロトタイプです。テンポ感の検証と運用イメージの確認用。

## 起動方法

1. `prototype/index.html` をダブルクリック(Chrome / Edge 推奨)
2. **F11** で全画面(キオスクモード)。1920×1080以外でも自動で等倍縮小されます
3. 完全オフラインで動作します

## 操作

| キー | 動作 |
|---|---|
| **スペース / Enter / クリック** | ゲームボタン(これ1つで全操作) |
| **1〜4**(待機画面中のみ) | 【スタッフ用】次プレイの結果ランクを強制指定(★1〜★4) |
| **5〜8**(待機画面中のみ) | 【スタッフ用】次リーチのゲームを強制指定(5=明媚 6=葵 7=蕎麦 8=溺惑) |
| **0**(待機画面中のみ) | 強制指定をすべてリセット(確率モードに戻す) |
| **D** | デバッグモード切替(押すと画面上部に「DEBUG ON/OFF」と表示されます) |

- 隠しキーは画面に一切表示されません(デバッグモード中のみ確認可)
- 強制指定は「次の1プレイ」のみ有効(ワンショット)

## ゲームフロー

1. **待機画面** → ボタンでスタート
2. **スロット**: ボタンでリールを左→中と停止
   - 2つ揃い → **リーチ!** → 担当メンバーのミニゲームへ
   - 揃わず → **復活チャレンジ**(勝てばスロットをもう一回/負けたら残念賞)
3. ミニゲーム成功度でランク決定。**大成功=3つ揃い(大当たり)** → ボーナス「4人まとめてキメろ!」で上乗せ
4. **結果画面** → 賞品ランク表示 → 自動で待機画面へ

## 調整方法(config.js)

`config.js` を書き換えて再読み込みするだけで反映されます。

- `slot.reachRate` … リーチ率(=ミニゲーム到達率)
- `minigames.revival.zoneShrinkPerRetry` / `speedUpPerRetry` … 復活チャレンジは回数無制限。挑戦ごとにゾーンが狭く・マーカーが速くなる度合い
- `minigames.*` … 各ゲームの判定ウィンドウ/ゾーン幅/速度(=実質当選率)
- `minigames.tokimeki.successRate` … ときめきトークの当選確率
- `ranks` / `bonusPrizes` … 賞品名の変更
- `layout` … タイトル・スロット筐体などの位置/スケール(Midjourney素材の構図ズレ対応)
- `members` … メンバー名・カラー・担当ゲームの割り当て

## フォント

- 使用フォント: **Yusei Magic**(基本・手書きマーカー)/ **Reggae One**(見出し・決め台詞の荒い叫び系)/ **DotGothic16**(リズムゲーのレトロ演出)
- 開発時はWeb(Google Fonts)から自動取得されます
- **オフライン本番前に必ず**: 上記3書体を [Google Fonts](https://fonts.google.com/) からダウンロードし、一度PCにインストールしてください(インストール済みならオフラインでも表示されます)。未インストール時は近い日本語フォントで代替表示されます

## コマ送りアニメ(キャラの動き)

- 待機画面の4人整列・スロット両サイドは `config.js` の `members[].anim` の画像を順番に切替表示(現在2枚。**3枚目ができたら配列に追加するだけ**)
- ジャスト・ビートの踊りは `minigames.beat.danceFrames`(差分を何枚でも追加可)
- 切替間隔は `anims.intervalMs`(ms)

## 素材の差し替え

- **サウンド**: `config.js` の `sounds` にファイルパスを指定(例 `{ file: 'assets/sounds/win.mp3' }`)。指定がない間は内蔵シンセ音が鳴ります。BGM(`bgmAttract` 等)はループ再生されます
- **画像**: `config.js` の `images` にファイル名を指定(`assets/images/` 内)。`null` の素材は仮図形で表示されます。メンバーの顔・立ち絵は `members` の `face` / `stand` / `standKime` で指定
- **リール窓の位置**: `config.js` の `layout.reels.centers` が `frame_slot.png`(1456×816)内の窓中心座標。筐体画像を差し替えたら、デバッグモード(Dキー)のグリッドを見ながらここを調整

## ミニゲームの追加(プラガブル設計)

1. `js/minigames/` に新しいJSファイルを作成
2. `MiniGames.register('ゲームID', { title: '表示名', async run(ctx) { ... return { level: 'perfect' } } })` を実装
   - `ctx.root`=シーン要素 / `ctx.cfg`=configの該当設定 / `ctx.bias`=補正('easy'/'hard'/null)
   - 戻り値 `level`: `perfect`(★4)/ `good`(★3)/ `miss`(★2)/ `fail`(★1)
3. `index.html` に `<script>` を1行追加
4. `config.js` の `members[].game` にIDを割り当て

## ファイル構成

```
prototype/
├ index.html          … 起動ファイル
├ config.js           … 全設定(確率・難易度・賞品・座標・音)
├ style.css           … 見た目(アメコミ調の仮スタイル)
└ js/
  ├ engine.js         … 基盤(画面・1ボタン入力・音・演出・デバッグ)
  ├ slot.js           … スロットパート
  ├ flow.js           … 全体進行(待機→スロット→ミニゲーム→結果)
  └ minigames/        … ミニゲーム(1ファイル=1ゲーム)
    ├ issen.js        … 一閃カウンター(A/反射神経)
    ├ beat.js         … ジャスト・ビート(B/リズム)
    ├ charge.js       … フルパワーチャージ(C/ゲージ)
    ├ tokimeki.js     … ときめきトーク(D/確率)
    ├ bonus.js        … 4人まとめてキメろ!(大当たり時ボーナス)
    └ revival.js      … 復活チャレンジ(リーチ不成立時)
```
